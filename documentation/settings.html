Setting up project

This document describes the connection and use of TailWindcss, setting up image optimization and styles.

File settings of the assembly template.config.js in the root of the project

Home Structure HTML components Conditions and conditional expressions Cychers that json

How to turn TailWindcss in the project:

must be specified tailwindcss: true

export default {
    tailwindcss: true,
    cleanCss: true,
    images: {
        ...
    },
 }

In the main style filesrc/scss/main.scss In the very top you need to proceed the following

@import "tailwindcss";

File tailwindcss.scss used to import TailWindcss styles and write components styles TailwindCSS

File tailwind.config.js that in the draft root is used to set up TailWindcss

CSS Cleaning in Build from the Disadvanty:

must be specified cleanCss: true

export default {
    tailwindcss: true,
    cleanCss: true,
    images: {
        ...
    },
 }

Optimization of images in Build:

To optimize in the original format specify imageMin: true.

To create Webp you need to specify imageMin: true та makeWebp: true. В ignore-webp The array is indicated classes for which not to create Webp.

To automatically fill the attributes width and height specify addImgSizes: true.

To adjust the quality of images to specify in imageQuality: { jpg: 80, png: [0.6, 0.8], webp: 80 }.

export default {
    ...
    images: {
        imageMin: true,
        makeWebp: true
        addImgSizes: true
        ignoreWebpClasses: ['ignore-webp'],
        imageQuality: {
            jpg: 80,
            png: [0.6, 0.8],
            webp: 80
        }
    },
}
