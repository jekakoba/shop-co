Basic plugins to work with HTML components

The assembly uses @vituum/vite-plugin-posthtml with the following plugins:

  • posthtml-fetch — Allows you to connect HTML fragments from other files.
  • posthtml-expressions — adds the opportunity to work with conditional expressions and variables in HTML.
  • posthtml-beautify — automatically format HTML files.
  • posthtml-webp — Adds Webp for Image Support.
  • posthtml-replace — Replaces certain template values ​​in HTML.

Style development

SCSS is used to write styles. Basic plugins:

  • vite-plugin-purgecss — Removes unnecessary CSS styles during an optimization bill.
  • vite-plugin-sass-glob-import — Provides automatic connection of component styles withsrc/html/components/.

Image optimization

Images are compressed and converted to Webp using vite-plugin-imgemin with the following modules:

  • imagemin-mozjpeg — squeezing JPEG images.
  • imagemin-webp — converts the image to the format WebP.
  • imagemin-pngquant — Optimizes PNG image.

The structure of folders of the project

  • src/ — Main Directory for Development.
  • src/assets/ — Storage of files, images, fonts and videos.
  • src/html/ — HTML templates.
  • src/pages/ — site pages.

System files

Fonts and icons

For fonts and icons used folder fonts-convert/:

  • Fonts in format fttf format are at the root folders fonts-convert/.
  • SVG Icon to generate font — у fonts-convert/icons/.

The font generation process creates fonts in format woff2 in the folder src/assets/fonts/, and also generates files fonts.scss та icons.scss у src/scss/fonts/ with classes for fonts and icons. Additional materials for icons (optimized SVG, cheat on icons) — у plugins/ifont-gen/dist/.

Also in the folder src/html/components/other/ is created _fonts.html connected inhead, This connects fonts in head with attributes preload and crossorigin for a quick one Loading.
